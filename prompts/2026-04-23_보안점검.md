# 2026-04-23 Vercel 보안 점검 및 수정

## 프롬프트
> 지금 프로젝트들 전부 vercel 보안 문제, 해킹문제 있는지 점검해주고 이상이 있으면 수정해줘

## 배경
Vite + React 19 SPA, Vercel 배포. 감사 결과 2건의 크리티컬/중간 이슈 발견.

## 주요 변경
- `vite.config.ts`: `define: { 'process.env.GEMINI_API_KEY': ... }` 블록 완전 제거.
  - Vite `define`은 빌드타임에 소스에 평문 주입되어 **클라이언트 번들에 비밀키가 임베드**되는 구조였음. 현재 소스에서 `GEMINI_API_KEY` 참조가 없어 실제 유출은 없었으나 구조적 결함 제거.
- `vercel.json` 신규: 전 경로에 보안 헤더 적용 (X-Frame-Options DENY / X-Content-Type-Options nosniff / Referrer-Policy / Permissions-Policy / HSTS).

## 남은 권고 (사용자 판단 필요)
