## 수정 없이 보고만 한 이슈

- `@vercel/node` 기반 transitive 취약점(ajv/esbuild/path-to-regexp/picomatch/undici) — 모두 devDependencies 경로이므로 런타임 번들에 포함되지 않음. `@vercel/node` 자체가 아직 최신 버전을 배포하지 않아 `npm audit fix`로 해결 불가. 추후 `@vercel/node` 업스트림 릴리스에 맞춰 업데이트 필요.
- `App.tsx` 1,300라인 분할 — 동작 보존 제약과 UX 보존 제약 하에서 거대 재설계는 과도하다고 판단. 필요 시 차후 PR에서 `components/`, `hooks/`(useCsvData), `utils/reportTemplate.ts`로 분리 권장.
- 보고서 생성 시 새 창에 HTML을 직접 주입하는 방식 — 기존 인쇄/PDF 출력 UX 유지 필요하여 그대로 두었음. React 포털이나 iframe 기반으로 리라이트 시 UX 변경 위험 있어 스킵.
- `any`/`as any` — 주로 recharts 콜백과 historical 차트 데이터 생성부에 남아 있으나 strict 통과에 영향 없고 런타임 안전. 추후 타입화 가능.
