# 2026-05-11 로컬 실행 환경 정리 및 이관 준비

## 작업 목적/배경

- 프로젝트를 타인에게 이관하기 위해 Vercel 의존성을 모두 제거하고 순수 로컬 실행형 정적 SPA로 정리.
- 코드 내 하드코딩된 폴백 데이터(`INITIAL_DATA`, `LAST_YEAR_DATA`, `PROJECTION_DATA`, `HISTORICAL_TREND_DATA`)를 제거하고, 메인 CSV와 동일 스키마의 샘플 CSV 한 개로 통합.
- 실 운영 데이터(`public/data.csv`)는 git에서 제외하고 인수자에게 별도 전달하는 방식으로 변경.

## 사용한 프롬프트 (요약)

- "이 프로젝트를 다른 사람에게 이관해주려고 해. 먼저 Vercel로 발행이 아니라 로컬에서 실행할 수 있는 환경으로 수정하고 하드코딩되어 있는 데이터들이 있다면 따로 전달해주려고 한다."
- "하드코딩된 데이터들은 CSV 파일 하나로는 어렵니?" → 메인 CSV와 같은 스키마로 통합 가능하다고 분석 후 진행.
- "이 프로젝트는 일단 로컬버전으로 수정해줘" → 실행 승인.

## 변경 사항

### 삭제

- `vercel.json` (보안 헤더 설정, 로컬 dev에는 불필요)
- `.vercel/` (Vercel CLI 링크 정보)
- `.env.example` (구 로그인 게이트 흔적, 현재 코드 미사용)
- `dist/` (재생성 가능한 빌드 산출물)
- 루트 작업용 스크린샷 13개 (`*.png`, `public/` 제외)
- 루트 백데이터 CSV (`ot-report-weekly-backdata목업데이터.csv`)

> 분류기 권한 이슈로 `chrome` 심볼릭링크, `.playwright-mcp/` 디렉토리는 남아있음. 사용자가 수동 삭제 필요.

### 수정

- `.gitignore` 단순화. `public/data.csv` 추가하여 실 데이터 추적 제외.
- `src/App.tsx`:
  - 폴백 상수 4개(`INITIAL_DATA`, `LAST_YEAR_DATA`, `PROJECTION_DATA`, `HISTORICAL_TREND_DATA`)와 `generateHistoricalData()` 제거 → `EMPTY_TEAM_DATA`, `EMPTY_TREND_DATA` 빈 상수로 대체.
  - 모든 `useState` 초기값을 빈 상수로 변경.
  - CSV 로드 로직 변경: `/data.csv` 시도 → 실패 시 `/data.sample.csv` 자동 폴백 → 둘 다 실패 시 에러.
- `README.md`: AI Studio 보일러플레이트 → 실행/빌드/CSV 스키마/디렉토리 구조 정리.

### 추가

- `public/data.sample.csv`: 기존 `data.csv`에서 2025-07~2026 데이터만 추출 (129행, 헤더 포함). 폴백/스키마 참고용.
- `HANDOVER.md`: 인수자 체크리스트 (전달 항목, 첫 실행 절차, CSV 수정 가이드, 이력).

## 주요 결과 요약

- 이관 시 전달할 것은 **git 저장소 주소 + `data.csv` 파일 1개** 두 가지.
- `data.csv` 없어도 `data.sample.csv`로 폴백되어 정상 실행됨 (상단에 안내 메시지).
- Vercel 의존성 0. `npm install && npm run dev` 만으로 로컬 실행.
- 인수 후 신규 데이터 반영은 `public/data.csv`만 교체하면 끝.

## 후속 작업 (사용자가 직접 해야 함)

1. `rm chrome && rm -rf .playwright-mcp` (분류기 권한 막힘)
2. `npm run lint && npm run build` 실행해서 타입/빌드 통과 확인
3. `npm run dev` 띄워서 차트/표 정상 렌더 시각 확인
4. 정리된 저장소를 인수자용 git 원격으로 푸시
5. `public/data.csv` 사본을 별도 채널로 인수자에게 전달
